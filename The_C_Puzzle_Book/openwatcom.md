# 🔍 16bit環境でCがどう違うのか

## ① `int` が16bitになる

現代（32/64bit）：

* `int` = 32bit

16bit環境（例：Open Watcom）：

* `int` = **16bit**

👉 影響：

```c
int x = 70000;  // オーバーフロー
```

---

## ② ポインタが“near / far”に分かれる

16bit特有の超重要ポイント。

### メモリモデル

* nearポインタ → 16bit（同一セグメント）
* farポインタ → 32bit（セグメント:オフセット）

```c
char *p;        // near（デフォルトの場合あり）
char far *fp;   // farポインタ
```

👉 つまり：

* ポインタサイズが統一されていない
* `sizeof(void*)` が環境依存

---

## ③ セグメント制約（64KBの壁）

16bit CPU（特にx86 real mode）は：

* 1セグメント = 64KB

👉 問題：

```c
char big[70000]; // コンパイルエラー or 分割必要
```

---

## ④ メモリモデルの概念

コンパイラ（特に Open Watcom）では：

| モデル    | 特徴               |
| ------ | ---------------- |
| small  | コード・データとも64KB以内  |
| medium | コードfar / データnear |
| large  | 両方far            |
| huge   | 配列が64KB超OK（特殊処理） |

👉 同じコードでも挙動が変わる

---

## ⑤ `sizeof` の結果が全然違う

例：

```c
sizeof(int)   // 2
sizeof(long)  // 4
sizeof(void*) // 2 or 4
```

👉 今の感覚で書くと事故る

---

## ⑥ スタックが極端に小さい

* 数KB〜数十KB程度

👉 影響：

* 再帰が危険
* 大きいローカル変数NG

---

## ⑦ 標準ライブラリの制約

* `malloc` も64KB制限
* ファイル操作も簡素

## 🔧 ① インストール

## 📁 ② ゲーム用フォルダを作る

例として、Cドライブにフォルダを作成：

```
C:\DOSGAMES
```

その中に遊びたいDOSゲームを入れます。

## 💻 ③ DOSBoxを起動

起動すると黒い画面（コマンドライン）が表示されます。

---

## 🔗 ④ フォルダをマウント（重要）

DOSBox内で、さっきのフォルダを仮想ドライブとして登録します：

```
mount c c:\dosgames
c:
```

👉 これで「Cドライブ」として使えるようになります。

---

## ▶️ ⑤ ゲームを起動

例えばゲームフォルダが `doom` の場合：

```
cd doom
dir
```

実行ファイル（例：`doom.exe` や `setup.exe`）を確認して：

```
doom
```

または

```
setup
```

---

## ⚙️ ⑥ サウンド設定（初回のみ推奨）

多くのゲームは最初に `setup.exe` を実行して：

* Sound Blaster（通常これでOK）
* Port: 220
* IRQ: 7
* DMA: 1

を選びます。

---

## 🚀 ⑦ 自動起動設定（便利）

毎回コマンドを打つのが面倒な場合：

1. 設定ファイル（`dosbox.conf`）を開く
2. 一番下の `[autoexec]` に追加：

```
mount c c:\dosgames
c:
cd doom
doom
```

👉 起動時に自動でゲーム開始

---

## 🎮 便利ショートカット

* フルスクリーン：`Alt + Enter`
* CPU速度調整：`Ctrl + F11 / F12`
* 終了：`Ctrl + F9`

## 💡 よくあるトラブル

* **日本語が文字化け** → 日本語対応版DOSBoxやフォント設定が必要
* **音が出ない** → setup.exeでSound Blasterに設定

# 🧰 Open Watcom の基本セットアップ（DOSBox想定）

## ① Open Watcom を入手・インストール

1. 公式サイトから **Open Watcom** をダウンロード
2. Windowsにインストール（例：`C:\WATCOM`）

---

## 📁 ② DOSBox用にフォルダをマウント

DOSBoxで以下を実行：

```id="wat1"
mount c c:\watcom
mount d c:\dosgames
c:
```

※ `C:` → Open Watcom
※ `D:` → 作業用（ソースコードなど）

---

## ⚙️ ③ 環境変数の設定（重要）

Open Watcomを使うには初期化が必要です。

```id="wat2"
cd \watcom\binw
owsetenv.bat
```

👉 これでコンパイラが使える状態になります

---

## ✍️ ④ 簡単なCプログラムを作る

`D:\HELLO` フォルダを作って、`hello.c` を用意：

```c
#include <stdio.h>

int main() {
    printf("Hello, DOS!\n");
    return 0;
}
```

---

## 🛠️ ⑤ コンパイル

DOSBox内で：

```id="wat3"
d:
cd hello
wcl hello.c
```

👉 `HELLO.EXE` が生成されます

---

## ▶️ ⑥ 実行

```id="wat4"
hello
```

---

# 🎯 DOS向けにビルドする（重要ポイント）

Open Watcomは複数ターゲットがあるので、DOS用にする場合：

```id="wat5"
wcl -bt=dos hello.c
```

主なオプション：

* `-bt=dos` → DOSターゲット
* `-l=dos` → DOS用ライブラリ

---

# 🧠 便利Tips

## ▶ フルスクリーンDOSアプリ

```id="wat6"
wcl -bt=dos -l=dos -ml hello.c
```

## ▶ 16bit / 32bit

* デフォルト：16bit（リアルモード）
* DOS/4GW（32bit）も対応可能

# ⚠️ よくあるつまずき

### ❌ コマンドが見つからない

→ `owsetenv.bat` を実行してない

### ❌ 日本語表示がおかしい

→ フォント or 日本語対応DOSBoxが必要

### ❌ EXEが動かない

→ DOS用でビルドしてない（`-bt=dos`必須）
